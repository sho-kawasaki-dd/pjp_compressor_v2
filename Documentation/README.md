# フォルダ一括圧縮アプリ（PDF・画像対応版）

PDF、JPG、PNGファイルを「フォルダ単位」で一括圧縮できるGUIアプリケーションです。
Ghostscriptがある環境では高品質な PDF 圧縮を行い、無い場合は pypdfによる圧縮へ自動フォールバックします。画像はPillowによる品質指定圧縮、PNGはpngquant（不可逆）も選択可能。さらに画像の一括リサイズ（手動指定 / 長辺指定、アスペクト比保持）にも対応。並列処理により大量ファイルでも高速に処理できます。

本アプリはモジュール分割され、GUIと圧縮ロジック、設定、サウンドユーティリティを分離しています。起動は `compressor_launcher.py` から行います（`ui_app.py` の `App` を起動）。

## 主な機能

- PDF 圧縮（Ghostscript推奨、pypdfフォールバック）
  - PDF圧縮エンジンは固定方針（Ghostscript優先 → pypdf → 圧縮不可）
    - システムに Ghostscriptが存在すれば Ghostscriptを使用（プリセット選択可）
    - Ghostscriptが無い場合のみpypdfを使用（簡易圧縮）
    - 両方無い場合は PDF圧縮不可（UIに状態表示）
    - エンジンの再検出はアプリ再起動にて行う仕様
  - Ghostscriptプリセット選択（/screen /ebook /printer /prepress /default）
  - PDF品質スライダー（任意）：ON時はGhostscript使用時にプリセットの画像DPIを上書き
- 画像圧縮
  - JPG/PNGをPillowで圧縮（品質0-100）
  - PNGをpngquantで不可逆圧縮（quality範囲自動設定）
  - 画像リサイズ（任意）：
    - モード選択（手動: 幅/高さ指定、または 長辺指定）
    - 長辺指定はプリセットドロップダウン（640/800/1024/1280/1600/1920/2048/2560/3840）から選択可能（任意入力も可）
    - アスペクト比保持の選択
    - パフォーマンス最適化：幅/高さの入力は処理開始時に一度だけ数値化し、空欄・非数値は0に自動補正（入力時の逐次検証は行わずUIを軽量化）
- 高速・安定
  - 圧縮前に入力フォルダ内のZIPアーカイブを自動的に再帰展開（最大25サイクルで打ち切り、循環ループを防止）
  - ThreadPoolExecutorによる並列処理
  - Before/Afterのファイルサイズをログに表示
  - 処理後に合計削減量と削減率を統計表示（PDF/JPG/PNGなど実際に圧縮を実施したファイルのみを集計）
  - CSVログ出力（任意）：各ファイルの処理結果を `compression_log_YYYYMMDD_HHMMSS.csv` に保存（入力/出力パス、拡張子、処理方法、サイズ、削減量/率）
  - 大量ファイルでもUIが固まらないように処理は別スレッドで実行
- 使いやすいUI
  - 入力フォルダのドラッグ＆ドロップ
  - 入出力フォルダのクリーンアップ（.pdf/.jpg/.jpeg/.png 限定、サブフォルダ含む。出力フォルダは.csvも対象）
  - 進捗バーと詳細ログ
  - 入力/出力フォルダの重なりを検知して自動リセット（警告とログを出力）
  - PDFエンジンの状態表示（Ghostscript / pypdf / 圧縮不可）をラベルで表示（再検出はアプリケーション再起動によって行う）
  - Ghostscript未検出時のUI自動制御
    - Ghostscriptが無い場合、PDFプリセット選択とPDF品質スライダーは自動的に無効化（グレーアウト）されます
  - リサイズ UI の動的無効化（視覚的にグレー化）
    - アプリ起動時は「画像を一括リサイズする」がOFFのため、幅/高さの入力、アスペクト比保持、手動/長辺指定ラジオ、長辺指定ドロップダウンはグレーアウト
    - 画像リサイズがOFFの場合、プリセットラジオと長辺指定ドロップダウンを無効化
    - 長辺指定モードでは、幅/高さ/アスペクト比保持の手動入力を無効化
    - 手動モードでは、長辺指定ドロップダウンを無効化
  - pngquantトグルの自動無効化
    - システムに `pngquant` が無い場合、トグルは無効化され「未検出のため無効」と表示

## 動作環境と依存

- Python 3.8+
- 必須: Pillow
- 推奨: Ghostscript（PDF高品質圧縮）, pypdf（Ghostscriptが無い場合の代替）
- 任意: pngquant（PNG不可逆圧縮）, tkinterdnd2（ドラッグ＆ドロップ）, pygame（サウンド再生）

仮想環境の作成を推奨します（PowerShell の例）:

カレントディレクトリを `pdf_photo_compressor` フォルダに設定してから次のコマンドを入力してください。

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install Pillow pypdf tkinterdnd2 pygame
```

requirements.txt を利用する場合:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

システムツールのインストール:

PCのOSがWindowsの場合は、CLIパッケージ管理ツールChocolateyを使うのがおすすめです。

- pngquant: Windows: `choco install pngquant` / macOS: `brew install pngquant` / Ubuntu/Debian: `sudo apt install pngquant`
- Ghostscript: Windows: `choco install ghostscript` / macOS: `brew install ghostscript` / Ubuntu/Debian: `sudo apt install ghostscript`

## 使い方（手順）

1. アプリを起動
     - カレントディレクトリをアプリフォルダにしてから仮想環境に入って、`python compressor_launcher.py`
     - 仮想環境が用意されていれば、compressor_launcher.ps1をダブルクリックしても起動可能(PowerShellスクリプト)
2. 入力フォルダを指定
     - 参照ボタンまたはドラッグ＆ドロップ
3. 出力フォルダを指定
     - Windows は初回起動時にデスクトップへ「圧縮済みファイル」フォルダ作成を提案
4. 圧縮設定
     - PDF: UIに表示される適用エンジン（Ghostscript優先）を確認し、Ghostscriptの場合はプリセットを選択
       - 必要に応じて「PDF品質スライダー」をONにし、DPIを調整（Ghostscript使用時にプリセットの画像DPIを上書き）
     - 画像: JPG/PNGの品質（0-100）、PNGをpngquantで圧縮するか選択
       - 画像リサイズを ON にして、モード（手動/長辺）とサイズ、アスペクト比保持を設定
       - 幅/高さが空欄や非数値の場合は自動で0に補正されます（処理開始時に一度だけ判定）。
       - リサイズ設定値の小数点以下は処理開始時に切り捨てられます。
5. 圧縮開始
     - 入力フォルダ内にZIPファイルがある場合、圧縮処理の前に自動的に展開されます（最大25サイクル、循環参照防止機能付き）
     - 進捗バー・ログで確認
     - ファイルごとの Before/After サイズをログに表示
     - 出力フォルダ内にCSVログ（`compression_log_YYYYMMDD_HHMMSS.csv`）が作成されます
6. 統計の確認
     - 合計サイズ、削減量、削減率を上部ラベルに表示（未対応拡張子など圧縮対象外のファイルは統計に含まれません）
7. クリーンアップ（必要に応じて）
     - 入力/出力フォルダの .pdf/.jpg/.jpeg/.png を削除（サブフォルダ含む。出力フォルダは.csvも対象）
     - 実行前に対象件数と注意書きを表示

## モジュール構成

- `compressor_launcher.py`: アプリの起動スクリプト（`App` を起動して mainloop 開始）
- `ui_app.py`: TkinterベースのGUI。圧縮処理は `compressor_utils` の関数を呼び出して実行
- `compressor_utils.py`: 圧縮ロジック・CSV ログ・クリーンアップなど（GUI 非依存）
- `configs.py`: 共有定数（Ghostscriptなどのプリセット、GUI既定値）
- `sound_utils.py`: pygameを使った効果音再生（任意依存・初期化失敗時は警告のみ）

## よくある質問

- Q: Ghostscriptとpypdf のどちらが良いですか？
  - A: Ghostscriptが高品質かつ柔軟で推奨です。pypdfはGhostscriptが使えない環境の簡易代替です。おそらくほとんどの場合、pypdfでは満足な圧縮効果が得られません。
- Q: PDFの品質スライダーは何をしますか？
  - A: ONにすると、GhostscriptでPDFを圧縮する際にプリセットの画像 DPI を上書きします（Color/Gray/Monoの各画像解像度に適用）。
- Q: PNG を大幅に圧縮したいです。どのように設定すればよいですか？
  - A: pngquantでの圧縮をONにしてください。品質範囲（min-max）はスライダーの値をもとに設定されます（max=スライダー値、min=max-20）。
- Q: 画像のリサイズ設定はどのように動きますか？
  - A: 手動モードでは幅/高さを指定可能、アスペクト比を保持するかどうかを選べます。アスペクト比を保持する場合は、片方の数値の入力で十分です。（もう一方は0でOK。）両方入力した場合は、小さいほうのピクセル数に合わせます。長辺モードでは長辺がドロップダウンの指定ピクセルになるように等比縮小します。
    - 幅/高さが空欄や非数値の場合は自動で0に補正されます（処理開始時に一度だけ判定）。また、設定値の小数点以下は処理開始時に切り捨てられます。
    - 手動モードで幅/高さの両方を0にした場合はリサイズされません。長辺モードで長辺を0にした場合もリサイズされません。
- Q: CSVログには何が記録されますか？
  - A: 各ファイルの入力/出力パス、拡張子、処理方法（Ghostscript/pypdf/Pillow/pngquant/無処理）、元サイズ、出力サイズ、削減量、削減率が記録されます。
- Q: CSVログの無効化や保存先の変更はできますか？
  - A: できます。設定画面の「CSVログを出力する」のチェックを外すと無効化。保存先を変更する場合はその右の入力欄にパスを直接入力するか「参照」ボタンから選択してください。未指定の場合は出力フォルダにタイムスタンプ付きファイルが作成されます。
- Q: クリーンアップで何が削除されますか？
  - A: .pdf/.jpg/.jpeg/.png が対象です。（**出力フォルダはログ消去のために.csvも対象**）サブフォルダ内の該当ファイルも削除されます。取り消し不可のため慎重に実行してください。

## 注意事項

- 大量のファイルを扱う場合、並列処理で高速化しますが、ディスク I/O によるボトルネックは発生し得ます。
- PDFはGhostscript のプリセットで圧縮します。PDF品質スライダーをONにした場合は、Ghostscript 使用時にプリセットの画像 DPIを上書きします。
- PDF圧縮エンジンは固定方針（Ghostscript優先 → pypdf → 圧縮不可）。圧縮エンジンの再検出はアプリ再起動にて行う仕様です。
- Ghostscriptが未検出の環境では、PDFプリセット選択とPDF品質スライダーは自動的に無効化（グレーアウト）されます。
- pngquantやGhostscript は**外部ソフトウェアとしてシステムにインストールする必要があります**（pythonのpipモジュールではインストールできません）。
- 入力フォルダと出力フォルダが重なる場合は、ファイル破損を避けるため自動的に両方ともデフォルトフォルダにリセットされ、警告とログが出ます。

## 開発者向けメモ

- 入力検証の最適化: リサイズの幅/高さは処理開始時に一度だけ文字列から数値化し、空欄・非数値は0に補正。入力時の逐次検証を省きUIイベント負荷を低減。
- 並列処理: `compress_folder()` は `ThreadPoolExecutor` を用いてファイル単位で並列化（`max(4, cpu_count())`）。大量ファイルでスループット改善。
- I/O ボトルネックについて: 画像/PNG/PDFの読み書き速度はストレージのディスク I/O 能力に依存します。CPUスレッドの並列度を上げてもディスク性能がボトルネックになる場合があります。
- UI分離: 圧縮はバックグラウンドスレッドで実行し、Tkのメインスレッドはログと進捗更新のみを担当（UIのフリーズ防止）。
- 実行時外部ツール検出と例外処理: Ghostscript/pngquantは実行時に `shutil.which` で検出し、未導入でも例外を致命化せずフォールバック（pypdf及びPillow）。
- PDF圧縮エンジン固定方針: 実行時にGhostscriptを最優先。未検出時のみpypdfを使用。圧縮エンジンの再検出はアプリ再起動にて行う仕様です。
- CSVログ: 1回のオープンで逐次追記し、行ごとにフラッシュを強制しない設計（過度な同期 I/O を回避）。
- 画像リサイズの計算: 長辺指定時の比率計算は軽量で、Pillowのリサイズ/保存が主なコスト。不要な再計算を避けるため設定を一度だけ作成。

備考:

- GUIと圧縮ロジックは分離され、`compressor_utils` は GUI非依存の関数群として再利用可能。
- 共有定数（Ghostscript プリセット、既定フォルダなど）は `configs.py` に集約。
- 効果音の操作関数は `sound_utils.py` に切り出し、pygameが無い環境でも動作継続可能（初期化失敗時は警告のみ）。
- アプリ起動時はリサイズ関連UIを無効化してグレーアウト。
